请回答1988

2020-05-20

最近刚看的一部韩剧，我个人觉得超好看，几乎每一集都有那个戳到你心里的点，整部剧每一个人都很暖，太真实了无论是情节还是人物。
百度：《请回答1988》刻画了5个家庭和10余个主要人物，人物戏份平均，谁都不是绝对主角；情感浓度也很均匀，谁也不是人生输赢家。这部剧不依靠高颜值的演员，没有大喜大悲的人物情感，没有跌宕起伏的人物命运，甚至也没有强烈的情节和反转，但该剧征服了很多观众的心。

“爸爸也是第一次做爸爸啊”

“神不能无处不在，所以创造了妈妈”

“亲不如邻”：谁家做了什么好吃的，整条街上的人都能尝到。有困难了大家一起帮着想办法，开心了就聚到一起吃吃饭唱唱歌打打花牌。不开心了就喝喝烧酒对着诉苦。

“双门洞小伙伴们的友情”

每周末都聚在一个房间里吃饭看电影，关系好到可以盖一个被子。

稳重踏实的学霸善宇，幽默睿智的娃娃鱼，善良单纯的学渣德善，毒舌但温暖的正焕还有可爱的阿泽。

德善

第一集，早上，宝拉要吃荷包蛋，余晖也要吃荷包蛋，德善没有说，德善妈妈拉开冰箱，发现只有两个荷包蛋

这个女孩懂事得让人心疼

德善正焕

正焕其实只是嘴上硬，对德善也很好。

每天早上假装系鞋带等德善上学。
公交车颠簸厉害就双臂护住德善。
德善去读书室回家很晚他不放心就一直等。
正焕只能看着阿泽出现在德善面前，而自己因为三个红灯，失去了那个机会。他终于明白了不是缘分造成的，是时机。因为自己数不清的犹豫，他永远与缘分失之交臂，失去了一个爱了多年的女孩。

德善阿泽

在网上看到一句，说阿泽才是对“知世故而不世故，才是最善良的成熟”。

阿泽一直是被保护良好的“双门洞纪念物”，不会扣扣子娃娃鱼帮他扣好，不会系鞋带狗焕帮他系好，不会开酸奶德善帮他开好，不会用筷子身边朋友帮忙把菜夹好，出去比赛爸爸帮他打点好。德善朋友偶尔来玩，想去看看阿泽，德善一口回绝：“阿泽不行。”
看起来天然无害的小朋友，内心却充满着极强的胜负欲，死也不想当第二，他清楚而明白地知道自己到底想要什么，朋友们开玩笑问他喜不喜欢德善，他目光坚定，斩钉截铁：“我喜欢德善，不是作为朋友，而是作为女人喜欢她。”

HEXO搭建个人博客教程

2020-05-20

“所谓博客，都是孤芳自赏”

利用hexo+github搭建博客，简单省钱不用买域名哈哈，然后记录一下搭建的过程，都是在b站还有一些大佬的博客里学的（网课老师讲的不认真听，花里胡哨倒是学的乐此不彼⁄(⁄ ⁄•⁄ω⁄•⁄ ⁄)⁄），后面会放链接，就不一一列出了。

准备(推荐用谷歌浏览器)

1.安装node.js https://nodejs.org/en/

2.下载安装git（官网下载安装）https://gitforwindows.org/

拥有GitHub账号https://github.com/

3.下载和安装hexo https://hexo.io/zh-cn/docs/

打开cmd（win+r cmd）并输入npm install hexo-cli -g

在桌面上新建一个空文件夹名为hexo

cmd安装完成之后打开Git bash

在cmd中进入hexo文件夹后输入$ hexo init

将博客与Github关联

1.在部署至github前先新建一个仓库，如blog。

​ 登录github，点头像、setting、Repositories

2.安装hexo-deployer-git：

3.修改_config.yml文件

4.生成静态文件并发布

注：如果在之前你有生成ssh公钥，那么就可以直接提交上去，如果没有，则需要填写邮箱以及密码。（生成ssh公钥见）https://www.jianshu.com/p/a7167e26b086

到这步基础框架大概就做好了，后面就是主题美化了

后面花里胡哨太多了，懒得写(/ω＼)直接看别人的吧

写博文

新建文章命令

_posts目录下新建.md文章（在编辑博文，Subline或者Typora）

生成静态文件: hexo g
提交到github: hexo d

搭建Hexo博客进阶篇–Hexo文档讲解（二）：

https://segmentfault.com/a/1190000009478129

关于图片素材

图片素材按官方教程说法，可统一放置在source/images目录中，并以 ![](/images/image.jpg) 方式引用,或者在 _config.yml 打开 post_asset_folder 功能，将当前文章所用的图片放置到source目录下的文章同名资源目录下，以 ![](image.jpg) 方式引用
